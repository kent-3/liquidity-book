  • append_permission_for_addr 🔒
    stores a PermissionKey {token_id: String, allowed_addr: String] for a given owner. Note that permission key is [owner, token_id, allowed_addr]. This function does not enforce that the list of PermissionKey stored is unique; while this doesn’t really matter, the ref implementation’s functions aim to ensure each entry is unique, for storage efficiency.
  • list_owner_permission_keys
    Return (Vec<PermissionKey { token_id, allowed_addr }>, u64) returns a list and total number of PermissionKeys for a given owner. The PermissionKeys represents (part of) the keys to retrieve all permissions an owner has currently granted
  • may_load_any_permission
